## 关键功能说明

### 1. 登录流程
1. 用户输入用户名和密码提交表单
2. 后端验证表单完整性
3. 查询数据库中的用户信息
4. 对用户输入的密码进行MD5加密后与数据库对比
5. 验证成功后，将用户ID、用户名、角色存入Session
6. 根据用户角色跳转到对应的首页

### 2. 权限控制使用方法
在其他蓝图的路由上直接使用装饰器即可：
```python
from app.utils.decorators import admin_required, teacher_required, student_required

# 管理员才能访问
@admin_bp.route('/students')
@admin_required
def student_list():
    students = Student.query.all()
    return render_template('admin/student_list.html', students=students)

# 教师才能访问
@teacher_bp.route('/courses')
@teacher_required
def my_courses():
    courses = Course.query.filter_by(teacher_id=session['user_id']).all()
    return render_template('teacher/my_courses.html', courses=courses)

# 学生才能访问
@student_bp.route('/grades')
@student_required
def my_grades():
    grades = Grade.query.filter_by(student_id=session['user_id']).all()
    return render_template('student/my_grades.html', grades=grades)
```

### 3. Session 中存储的信息
```python
session['user_id']    # 用户ID(users表的主键)
session['username']   # 登录用户名
session['role']       # 用户角色(admin/teacher/student)
```

### 4. 安全特性
- 密码使用MD5加密存储，不保存明文
- 所有受保护的路由都有登录验证和权限验证
- 登录成功后会清空之前的Session，防止会话固定攻击
- 表单提交有基本的验证
- 使用Bootstrap的alert组件显示友好的错误提示
