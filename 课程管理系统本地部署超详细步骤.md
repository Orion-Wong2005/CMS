# 课程管理系统 **Windows本地部署超详细步骤**
专为课程设计新手优化，**零遗漏、可直接照着操作**，从软件安装到系统运行全程截图级指引，全程约10分钟完成。

---

## 📋 前置说明
- **适用系统**：Windows 10/11（64位）
- **最终效果**：在你的电脑上运行完整的课程管理系统，可通过浏览器访问
- **默认账号**：管理员 `admin` / `123456`（部署完成后请立即修改）

---

## 🔧 第一步：安装必备软件（3个）
### 1. 安装Python 3.10（推荐版本，兼容性最好）
1.  下载地址：https://www.python.org/ftp/python/3.10.11/python-3.10.11-amd64.exe
2.  运行安装程序，**务必勾选最下方的 "Add Python 3.10 to PATH"**（非常重要！）
3.  点击 "Install Now" 完成安装
4.  验证安装：按 `Win+R` 输入 `cmd` 打开命令提示符，输入以下命令，显示版本号即成功
    ```bash
    python --version
    pip --version
    ```

### 2. 安装MySQL 8.0（数据库）
1.  下载地址：https://dev.mysql.com/get/Downloads/MySQLInstaller/mysql-installer-community-8.0.36.0.msi
2.  运行安装程序，选择 "Developer Default"（开发者默认），点击 Next
3.  一路点击 Next，直到 "Authentication Method" 页面，选择 **"Use Legacy Authentication Method"**（兼容旧版驱动）
4.  设置MySQL root用户密码（**请记住这个密码，后面会用到**），比如设为 `123456`
5.  一路点击 Next，最后点击 Execute 完成安装
6.  验证安装：在开始菜单找到 "MySQL 8.0 Command Line Client"，输入你设置的root密码，能进入MySQL命令行即成功

### 3. 安装Navicat（可选，数据库可视化工具）
- 下载地址：https://www.navicat.com/en/download/navicat-for-mysql
- 用于方便查看和管理数据库，也可以用MySQL自带的命令行

---

## 🗄️ 第二步：创建数据库
1.  打开MySQL命令行（开始菜单→MySQL 8.0 Command Line Client）
2.  输入你的root密码，登录MySQL
3.  复制粘贴以下命令，按回车执行（创建系统所需的数据库）
    ```sql
    CREATE DATABASE IF NOT EXISTS course_management_system 
    DEFAULT CHARACTER SET utf8mb4 
    DEFAULT COLLATE utf8mb4_unicode_ci;
    ```
4.  输入 `exit` 退出MySQL命令行

---

## 📂 第三步：准备项目代码
1.  将你之前写好的课程管理系统项目文件夹，放到一个**没有中文和空格**的路径下
    - ✅ 正确路径：`D:\projects\course_management_system`
    - ❌ 错误路径：`D:\我的项目\课程管理系统 v1.0`
2.  确认项目根目录下有以下核心文件：
    ```
    course_management_system/
    ├── app/                  # 应用代码目录
    ├── run.py                # 启动文件
    └── requirements.txt      # 依赖包列表
    ```

## 🐍 第四步：创建Python虚拟环境（必做）
虚拟环境可以隔离项目依赖，避免和其他项目冲突。

1.  打开项目文件夹，在地址栏输入 `cmd`，按回车（直接在当前目录打开命令提示符）
2.  输入以下命令创建虚拟环境
    ```bash
    python -m venv venv
    ```
3.  激活虚拟环境
    ```bash
    venv\Scripts\activate
    ```
4.  成功激活后，命令行前面会出现 `(venv)` 标识，如下所示：
    ```
    (venv) D:\projects\course_management_system>
    ```

---

## 📦 第五步：安装项目依赖
在激活虚拟环境的命令行中，输入以下命令安装所有需要的Python包：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```
- `-i https://pypi.tuna.tsinghua.edu.cn/simple` 使用清华镜像源，下载速度更快
- 等待所有包安装完成，没有报错即成功

---

## ⚙️ 第六步：修改数据库配置
1.  打开项目中的 `app/config.py` 文件
2.  修改数据库连接字符串，将 `你的MySQL密码` 替换为你第二步设置的root密码
    ```python
    import os

    class Config:
        # 密钥（本地开发可以不用改，生产环境必须修改）
        SECRET_KEY = os.environ.get('SECRET_KEY') or 'dev-secret-key-123456'
        
        # 数据库配置（只需要修改密码部分）
        SQLALCHEMY_DATABASE_URI = 'mysql+pymysql://root:你的MySQL密码@localhost:3306/course_management_system'
        SQLALCHEMY_TRACK_MODIFICATIONS = False
        
        # 本地开发开启调试模式
        DEBUG = True
    ```
3.  保存文件

---

## 🗃️ 第七步：初始化数据库表
这一步会自动在数据库中创建所有需要的表，并插入默认管理员账号。

在激活虚拟环境的命令行中，依次执行以下3条命令：
```bash
# 1. 初始化迁移环境（只需要执行一次）
flask db init

# 2. 生成数据库迁移脚本
flask db migrate -m "initial tables"

# 3. 应用迁移，创建所有表
flask db upgrade
```

✅ 验证：打开Navicat，连接本地MySQL，查看 `course_management_system` 数据库，应该能看到7张表（users、students、teachers、courses、enrollments、grades、schedules）

## ❌ 常见问题排查
### 问题1：`python` 不是内部或外部命令
- 原因：安装Python时没有勾选 "Add Python to PATH"
- 解决：重新运行Python安装程序，勾选 "Add Python to PATH"，然后重启电脑

### 问题2：数据库连接失败
- 原因1：MySQL服务没有启动
  - 解决：按 `Win+R` 输入 `services.msc`，找到 "MySQL80" 服务，右键启动
- 原因2：密码错误
  - 解决：确认 `app/config.py` 中的密码和你设置的MySQL root密码一致
- 原因3：3306端口被占用
  - 解决：关闭占用3306端口的程序，或者修改MySQL端口

### 问题3：`flask` 不是内部或外部命令
- 原因：没有激活虚拟环境，或者依赖没有安装成功
- 解决：重新执行 `venv\Scripts\activate` 激活虚拟环境，然后重新安装依赖

### 问题4：端口5000被占用
- 原因：其他程序占用了5000端口
- 解决：修改 `run.py` 文件，将端口改为其他未被占用的端口（比如5001）
  ```python
  if __name__ == '__main__':
      app.run(debug=True, host='0.0.0.0', port=5001)
  ```

## 📌 日常使用说明
1.  每次运行系统前，都需要先激活虚拟环境：
    ```bash
    cd D:\projects\course_management_system
    venv\Scripts\activate
    python run.py
    ```

3.  退出虚拟环境：输入 `deactivate`
